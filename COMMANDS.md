# 명령어

**재생 제어**
- `/music jump` - 대기열 항목 즉시 재생
- `/music remove` - 현재 곡 삭제
- `/music skip` - 다음 곡으로 스킵
- `/music pause` - 일시정지
- `/music resume` - 재개

**대기열 관리**
- `/music add` - 대기열에 추가
- `/music list` - 대기열 보기
- `/music np` - 현재 재생 중인 곡 확인
- `/music qdel` - 대기열 항목 삭제
- `/music qclear` - 대기열 전체 삭제
- `/music qshuffle` - 대기열 섞기
- `/music qdedupe` - 중복 제거
- `/music qmove` - 순서 변경

**채널 / 기타**
- `/music join` - 음성 채널 참가
- `/music leave` - 음성 채널 퇴장
- `/music help` - 명령어 목록
- `/music webui` - Web UI 주소
- `/version` - 버전 정보
- `/logs` - 서버 로그

> **참고:** 음성 채널에 연결되지 않은 상태에서도 대부분의 명령어를 사용할 수 있다. 웹 UI의 브라우저 재생 기능을 통해 음성 채널 없이도 음악을 들을 수 있다.


## `/music help`

사용 가능한 명령어 목록과 간략한 도움말을 출력한다. (본인에게만 보임)


## `/music join <CHANNEL>`

_예시:_ `/music join 음성채팅방`

파라미터:
- CHANNEL: (Optional) Discord 음성 채널 이름.

음성 채널에 봇이 참가한다. CHANNEL 파라미터로 봇이 참가할 채널 이름을 지정할 수 있다.

CHANNEL 파라미터 없이 실행시, 명령을 실행한 사용자와 동일한 음성 채널에 참가한다.

사용자가 음성 채널에 참가 중이 아닌 경우, 첫번째 음성 채널(서버 채널 목록 중 가장 위에 있는 채널)에 참가한다.

이미 재생 중인 곡이 있으면 새 연결에 자동으로 플레이어를 구독하여 음성 재생이 시작된다.

웹 UI에 토스트 알림이 전송된다.


## `/music add <URL>`

_예시:_ `/music add https://youtu.be/dQw4w9WgXcQ`

파라미터:
- URL: (Required) 음악/영상 URL. YouTube, SoundCloud 등 yt-dlp가 지원하는 모든 URL 사용 가능.

대기열의 마지막에 항목을 추가한다.

현재 봇이 음성 채널에 참가 중이 아닌 경우, 자동으로 음성 채널 참가를 시도한다. (실패해도 대기열 추가는 진행된다.)

URL이 올바른 음악 URL이 아닌 경우, 에러 메시지를 출력하고 아무 작업도 하지 않는다.

URL이 여러 항목을 포함하는 경우 (예: YouTube 재생 목록) 전체 항목을 대기열에 추가한다. 이 경우 기존 대기열 및 현재 재생 중인 곡과 중복되는 URL은 자동으로 제거된다.

각 항목에 대해 썸네일 URL이 자동으로 수집되며, 웹 UI 대기열에서 앨범 아트로 표시된다. YouTube 재생 목록의 경우 비디오 ID에서 자동 생성된다.

대기열에 추가만 수행하며, 자동 재생은 하지 않는다.

웹 UI에 토스트 알림이 전송된다.


## `/music list [hidden]`

파라미터:
- hidden: (Optional, 기본값: false) true로 설정하면 나에게만 보이는 메시지로 표시한다.

현재 재생 중인 항목과 대기열 내용을 embed로 출력한다.

재생 중인 항목이 있으면 경과 시간 / 총 재생 길이를 표시한다.

일시정지 중인 경우 노란색으로 표시한다.

대기열이 10개를 초과하는 경우 페이지 이동 버튼이 표시된다.

재생 제어 버튼(⏸ 일시정지 / ⏭ 건너뛰기 / ⏹ 중단)이 함께 표시된다.


## `/music np`

현재 재생 중인 항목을 본인에게만 보이는 메시지로 확인한다.

경과 시간 / 총 재생 길이, 대기 중인 곡 수, 재생 제어 버튼을 함께 표시한다.

재생 중인 항목이 없는 경우, 에러 메시지를 출력하고 아무 작업도 하지 않는다.


## `/music jump <INDEX>`

_예시:_ `/music jump 3`

파라미터:
- INDEX: (Optional) 대기열의 항목 인덱스.

대기열에서 INDEX 번째에 있는 항목을 제거하고 해당 음악을 즉시 재생한다.

INDEX 파라미터 없이 실행시, 대기열의 첫번째 항목이 선택된다.

대기열이 비어있거나, INDEX가 범위를 벗어나는 경우, 에러 메시지를 출력하고 아무 작업도 하지 않는다.

현재 이미 재생 중인 음악이 있는 경우에는 그 음악이 중단된다.


## `/music remove`

현재 재생 중인 곡을 삭제한다.

재생 중인 항목이 없는 경우 에러 메시지를 출력하고 아무 작업도 하지 않는다.

웹 UI에 토스트 알림이 전송된다.


## `/music skip`

현재 재생 중인 음악을 즉시 중단하고 대기열의 첫번째 항목을 재생한다.

건너뛴 항목은 대기열 맨 뒤로 이동한다.

다음 곡이 사전 다운로드되어 있으면 즉시 재생된다.

웹 UI에 토스트 알림이 전송된다.


## `/music pause`

현재 재생 중인 음악을 일시정지한다.

이미 일시정지 상태인 경우, 에러 메시지를 출력하고 아무 작업도 하지 않는다.

웹 UI에 토스트 알림이 전송되며, 재생 바가 회색으로 변경되고 CD 회전이 멈춘다.

Discord 봇 상태가 ⏸️ 일시정지로 변경된다.


## `/music resume`

일시정지된 음악을 재개한다.

일시정지 중 웹 UI에서 타임라인을 변경한 경우, 변경된 위치에서 재생을 시작한다.

일시정지 상태가 아닌 경우, 에러 메시지를 출력하고 아무 작업도 하지 않는다.

웹 UI에 토스트 알림이 전송된다.


## `/music qdel <INDEX>`

_예시:_ `/music qdel 3`

파라미터:
- INDEX: (Optional) 대기열의 항목 인덱스.

대기열에서 INDEX 번째에 있는 항목을 삭제한다.

대기열이 비어있거나, INDEX가 범위를 벗어나는 경우, 에러 메시지를 출력하고 아무 작업도 하지 않는다.

INDEX 파라미터 없이 실행시, 대기열에서 가장 마지막 항목을 삭제한다.

웹 UI에 토스트 알림이 전송된다.


## `/music qclear`

현재 대기열을 전부 삭제한다. 실행 시 확인 버튼이 표시되며, 확인 후 삭제된다.

(현재 재생 중인 음악에는 영향을 미치지 않는다.)

웹 UI에 토스트 알림 및 쓸려나감 애니메이션이 표시된다.


## `/music qshuffle`

대기열의 항목을 무작위 순서로 섞는다.

대기열이 비어있는 경우, 에러 메시지를 출력하고 아무 작업도 하지 않는다.

사전 다운로드 캐시가 자동으로 갱신된다.

웹 UI에 토스트 알림 및 흔들림 애니메이션이 표시된다.


## `/music qdedupe`

대기열에서 같은 URL이 중복된 항목을 제거한다. 먼저 등장한 항목을 유지한다.

중복 항목이 없는 경우, 그 사실을 알리고 아무 작업도 하지 않는다.

웹 UI에 토스트 알림 및 압축 애니메이션이 표시된다.


## `/music qmove <FROM> <TO>`

_예시:_ `/music qmove 3 1`

파라미터:
- FROM: (Required) 이동할 항목의 현재 인덱스 (1부터 시작).
- TO: (Required) 이동할 목표 인덱스 (1부터 시작).

대기열에서 FROM 번째 항목을 TO 번째 위치로 이동한다.

FROM 또는 TO가 범위를 벗어나거나, 대기열이 비어있는 경우, 에러 메시지를 출력하고 아무 작업도 하지 않는다.

사전 다운로드 캐시가 필요 시 자동으로 갱신된다.

웹 UI에 토스트 알림이 전송된다.


## `/music leave`

음성 채널에서 봇이 퇴장한다.

현재 재생 중인 음악은 유지된다. (음성 채널 연결만 해제)

봇이 음성 채널에 참가 중이 아닌 경우, 에러 메시지를 출력하고 아무 작업도 하지 않는다.

웹 UI에 토스트 알림이 전송된다.


## `/music webui`

설정된 Web UI 주소를 본인에게만 보이는 메시지로 표시한다.

`WEB_UI_URL` 환경변수가 설정된 경우 해당 값을, 설정되지 않은 경우 `http://localhost:{WEB_PORT}`를 표시한다.


## `/version`

_package.json_ 파일을 읽고 현재 봇의 버전과 설명을 출력한다.

현재 JavaScript 런타임과 의존성(yt-dlp, ffmpeg) 정보도 출력한다.


## `/logs [count] [level]`

파라미터:
- count: (Optional, 기본값: 20, 최대: 30) 표시할 로그 수.
- level: (Optional, 기본값: 전체) 로그 레벨 필터. `전체` / `INFO` / `WARN` / `ERROR`

최근 서버 로그를 본인에게만 보이는 메시지로 표시한다.

레벨에 따라 색상이 구분된다. (INFO: 파랑, WARN: 노랑, ERROR: 빨강)
